Zalety TDD :

  1. Wymusza na tobie dokładne przemyślenie potrzebnych klas,metod i właściwości
  2. Ustalasz CO kod powinnien robić, a nie JAK
  3. Możesz od razu testować fragment aplikacji

Zalety TDD :

  1. TDD wymusza na tobie modularność
  2. Co jeśli baza danych nie jest gotowa? Co jeśli WEB API nie jest gotowe?
  3. Przy każdej zmianie kodu możesz sprawdzić czy czegoś nie rozwaliłeś
  4. Testy stają się dokumentacja kodu

Zalety TDD :

  1. Unikasz przeinżeniorowania
  2. Fajnie by było gydby każda aplikacja Legacy miała testy
Cykl:
  RED   GREEN  REFACTOR
DEMO: live coding

AntyWzorce TDD:

  1. Zależność pomiędzy testami
  2. Testowanie detali implementacyjnych
  3. Powolne testy

Ograniczenia TDD:

  1. Możliwe dziury w testach
  2. TDD w samo sobie nie jest wystarczające

Aby mieć pewność, że wszystko jest w porządku, z twoją aplikacją musisz także

  1. Sprawdzać proces wdrażania samej aplikacji na serwer lub na chmurę
  2. Patrzyć na zmiany w sieci?
  3. Mieć testy integracyjne, czyli całościowe
Ograniczenia TDD:
Potrzeba wsparcia zarządu i innych programistów

Pytania o TDD:

  1. Czy Agile jest wymagany do TDD?
  2. Czy naprawdę muszę pisać testy najpierw?
